Canvas Curator

A unified Canvas LMS workspace that lets instructional designers audit, remediate, curate, and build course content — with WCAG 2.1 accessibility checks and AI-assisted authoring baked in from the first click.

View on GitHub

A collaborative build

Canvas Curator is a team effort. It's been built in tight collaboration with engineering and instructional design partners at ASU Online — pairing accessibility expertise, LMS workflow knowledge, and full-stack development. My contributions focus on the instructional design side: shaping the accessibility audit flow, the AI-generated alt text and rubric pipelines, the Create workflow for turning Word and PDF source docs into Canvas-ready courses, and usability testing with real faculty.

The tool is the direct product of dozens of touch-base meetings, pairing sessions, merge reports, and user-feedback cycles — documented transparently in the repo (handoff docs, merge reports, fix logs, planning notes) so every decision is traceable.

What it does

Curate

Connect to an existing Canvas course, pull every page, assignment, discussion, quiz, file, and module into a single workspace. Run a WCAG 2.1 audit against the full course. Edit content inline in a rich-text editor with accessibility linting. Push changes back to Canvas when ready.

Create

Upload Word or PDF source documents. AI extracts and structures the content. Apply a themed HTML template, review the generated modules and pages, and push a fully organized course straight to Canvas — in a fraction of the time manual authoring takes.

Transfer

Copy content between two Canvas courses using Canvas's native content migration API — with the broken-link and image-sibling repair that IDs actually need after a transfer, not just a raw copy.

TagFlow

A visual PDF-tagging canvas: draw zones for headings, paragraphs, figures, and tables directly on the page preview, then export as a properly tagged, accessible PDF. Remediation that used to need Adobe Acrobat Pro and a lot of patience.

Images

AI-generated alt text, one at a time or in batch. Perceptual-hash duplicate detection across an entire course. Long-description management for complex graphics. Images are a common accessibility blocker — Canvas Curator makes them a solved problem rather than a lingering ticket.

Reports

Download audit results and faculty review spreadsheets as Excel files, ready to share with department leads, course owners, and QA reviewers.

Accessibility is not an afterthought

Accessibility runs through every surface of the tool rather than being bolted on at the end:

  • WCAG 2.1 audits are first-class, run against pulled Canvas content, not run once and forgotten.
  • A11y Lint Panel flags issues in the rich-text editor as the author types — heading order, alt text, link text, color contrast, table headers.
  • AI Vision Panel inspects images in context and proposes alt text that's tied to what's actually happening in the image, not generic filler.
  • Ally API integration brings ASU's institutional accessibility engine into the workspace, so remediation and reporting stay aligned with enterprise standards.
  • TagFlow turns PDF structural tagging from a specialist task into a visual drag-and-draw workflow — IDs, not just a11y specialists, can ship tagged PDFs.
  • Remediation queue lets a designer work through all issues across a course in one pass, rather than jumping between Canvas pages one at a time.

Time it saves instructional designers

  • Course audits: what used to take a day of page-by-page checking with separate tools now runs as a single pull + audit, with every issue linked to the source element.
  • Alt text at scale: batch generation means an entire course of 200+ images can be drafted in minutes, with a reviewer pass instead of a from-scratch authoring pass.
  • Course creation: uploading a Word syllabus + faculty notes produces a structured Canvas course skeleton immediately — modules, pages, assignments pre-scaffolded against a template.
  • PDF remediation: visual tagging in TagFlow cuts out the Acrobat tag-tree learning curve; a designer can remediate a study guide in the same session they receive it.
  • Content transfers: the broken-link repair and image-sibling fixes eliminate the post-migration cleanup that otherwise eats hours per course.
  • Faculty review: reviewers get a focused spreadsheet instead of ad-hoc email threads, and feedback flows straight back into the tool.

Technical foundation

  • Frontend — React 19 + TypeScript, Vite, Tailwind, Zustand for state, Tiptap for rich-text editing (with ~12 custom extensions: accordions, callouts, resizable images, HTML embed blocks).
  • Backend — FastAPI (Python 3) with a hand-built Canvas REST client that handles pagination, retries, and rate limits against Canvas v1.
  • Storage — Cloudflare R2 for session state and files, Supabase Postgres for users and roles.
  • Auth — Google OAuth via Supabase, HS256 JWTs (7-day expiry).
  • AI routing — Tiered provider strategy: ASU AIML Platform (Claude Sonnet) → Anthropic Claude API → local Ollama as fallback, so the tool keeps working even if a provider is down or a user is air-gapped.
  • Accessibility engine — Ally API v1 for institutional-grade audits.
